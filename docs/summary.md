# 项目总结 · A Life in Songs

> 状态：**可本地预览，已推送 GitHub** — 核心内容与功能已就位，背景视频与 About 待补。  
> 最后更新：2026-06-15

## 项目目标

将 Dreamways 沉浸式网站框架定制为个人人生回忆录 **A Life in Songs**，以 18 首歌曲串联从童年到当下的真实经历。

## 已完成

### 内容与配置

- 18 章故事正文、角色卡文案全部写入 `config.ts`
- 站点重命名：A Life in Songs / 一首歌一种心情
- 区块标题人生主题化：一路歌行、故事里的人们、浅唱·留声·沉醉
- 7 个核心人物角色卡分配（小梦、阿驰、瑶瑶、小晴、小洁、娣总、东方波）

### 媒体资源

- 18 首章节 BGM（`public/audio/`）
- 18 张章节插画（`ch01.png` … `ch18.png`）
- 氛围背景音乐 The Deep South
- 首屏 `首屏.mp4`、第 1 章 `ch01.mp4`、第 7 章 `ch07.mp4`
- 角色卡正式图（`ch01.png` … `ch18.png`）与真人参考图（`ref/`）

### 功能实现

- **章节故事区**：小图肖像 + 角色简介 + 正文 + 点击播放 BGM
- **角色卡区**：缩小尺寸、分页（6×3）、翻转详情
- **音频互斥**：章节歌 / 角色语音 / 氛围 BGM 全局只播一条
- **氛围音乐**：导航栏开关，默认开启，与章节歌互斥
- **背景视频**：第 1、7 章专属视频；其余章节黑底（不滥用首屏视频）
- **性能**：移除曾导致 Chrome 卡死的全站自动 BGM + 重负载延后逻辑

## 风险与注意

- **版权**：18 首流行歌 + 氛围曲公开部署需自行评估
- **隐私**：故事涉及真实人物，公开前需脱敏确认
- **网络**：国内访问 GitHub 推送可能需代理
- **性能**：18 章 ScrollTrigger 在低端机仍可能偏重，避免再加自动预载大量媒体

## 阶段评价

项目已从「文案准备」进入 **可演示成品** 阶段：打开网站可浏览 18 章故事、听章节歌与氛围音乐、翻看角色卡。距离「完整视觉体验」主要差背景视频批量补全与 About 区块；距离「对外发布」差部署方案与版权确认。

## 2026-06-16 补充总结

近期围绕 Zeabur 预览站的移动端加载慢、卡片点击后文字不稳定显示做了专项优化。

- 新增并接入章节视频：`ch02.mp4`、`ch03.mp4`、`ch12.mp4`；第 7 章视频统一使用 `ch07.mp4`，不再使用 `ch07.trim.mp4` 路径。
- 背景视频加载改为 poster 优先、视频 metadata 预加载、视频就绪后淡入，降低打开页面时的资源压力。
- 增加移动端视频白名单 `mobileVideoIds`，手机端只启用少量关键章节视频，避免 18 个章节全部做成 MP4 后拖慢首屏和交互。
- `Companions` 在移动端使用平面翻面逻辑，不依赖 3D transform 的文字渲染，提升微信/小红书/移动浏览器内的可读性。
- 角色语音音频延迟挂载，点击播放时再加载，减少初始 DOM 与网络负担。
- 轻量背景 poster 使用 `public/images/thumbs/chXX.webp`，用于替代 2.5MB-3MB 级别原始 PNG 参与背景首屏显示。

后续资源策略建议继续维持“少量关键视频 + 全章节轻量 poster + 用户触发音频”的模式。即使未来补齐 18 个场景 MP4，也不建议首屏或移动端全部预加载。
