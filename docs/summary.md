# 项目总结 · 我的一生（阶段一）

> 状态：**进行中** — 故事文案已完成，资源配置与 config 写入待进行。  
> 最后更新：2026-06-10

## 项目目标

将 Dreamways 沉浸式网站框架定制为个人人生回忆录，以 18 首歌曲串联从童年到当下的真实经历，供本人回顾与展示。

## 已完成

### 需求与内容

- 明确定制方向：人生旅途 × 歌曲记忆
- 完成 **18 章故事正文**，含多轮用户事实修订
- 建立章节 id / 标题 / 时期 / 歌曲速查表
- 梳理每章待填 config 字段与媒体清单

### 文档体系

在 `docs/` 建立完整项目管理文档：plan、readme、standard、talk、summary、review、document。

## 关键技术点

- **配置驱动**：业务内容集中于 `src/content/config.ts`
- **双区块联动**：每章 `chapters[]` 同时驱动 Seasons 故事面板与 Companions 角色卡
- **滚动叙事**：GSAP ScrollTrigger 切换章节视频与状态
- **18 章规模**：框架原设计 4 章四季，现扩展至 18 章，布局自动适配但需关注性能与浏览长度

## 需求细节摘要

- 第 1 章核心记忆：同班青梅编舞领舞《大地飞歌》，用户台下观看
- 大学前三章顺序：只对你有感觉 → 想象之中 → 飘摇
- 感情线：女友从复读到研究生，与《匆匆那年》及九夜茴小说关联
- 考研线：大四《之子于归》+ 200 元手机 + 图书馆备考
- 最新章：移植仓内《惊鸿醉》与浪浪山、不放弃精神

## 风险与注意

- **版权**：18 首流行歌曲若公开部署需考虑 BGM 方案
- **体量**：18 章内容较长，访客需长时间滚动；可考虑后续加章节导航
- **隐私**：涉及女友、同学等，公开前需用户自行脱敏确认
- **硬编码**：部分 UI 文案不在 config，需少量改组件

## 阶段评价

故事层已具备上线雏形；距可预览网站差「角色卡 + 站点文案 + 媒体资源 + config 注入」四步。建议下一步先完成角色卡与 siteConfig 文字（可无媒体占位），再批量准备资源。
