# 项目规范 · A Life in Songs

## 1. 文档规范

- 所有项目文档放在 `docs/` 目录
- 需求变更必须记录到 `plan.md` 的「需求变更记录」
- 内容或配置更新后同步更新 `document.md` 索引
- 对话与技术细节记录到 `talk.md`
- 阶段结束更新 `summary.md`、`review.md`

## 2. 内容规范

### 故事文案

- 语气：第一人称回忆录，真实、克制、有画面感
- 单章 `story` 建议 150–280 字；过长可在填入 config 时精简
- 不虚构用户未提及的关键事实；用户修订以 `我的一生-故事文案.md` 为准

### 章节配置

- `id`：全小写英文 + 连字符，全局唯一，如 `college-dance`
- `title`：建议格式 `第X章 · 时期 · 歌曲名`
- 无音频时 `audio: ''`，不写占位路径
- 无角色语音时 `character.audio: ''`

### 角色卡片

- `name` / `role`：简短，2–8 字为宜
- `desc`：正面一句话，≤ 30 字
- `detail` / `intro`：可略长，与当章故事一致，不矛盾
- `emoji`：图片加载失败时的备用显示
- `image`：建议正方形或 3:4，路径以 `/images/` 开头

### 网站文案

- `siteConfig.title`：主标题，简短有力
- `heroSubtitle`：一句话概括整站主题
- `footerQuote`：一句人生感悟，可加书名号式引号

## 3. 资源规范

### 音频

- 章节 BGM、角色语音、氛围音乐通过 `audioManager` **全局互斥**，新增播放器须接入 `playExclusive` / `stopExclusive`
- 中文或含空格路径使用 `encodeURI` 后再赋给 `<audio src>`
- 无音频时 `audio: ''`，UI 自动禁用
- 氛围音乐配置在 `siteConfig.ambientMusic`，勿与章节歌硬编码冲突

### 视频

- 格式：`.mp4`（H.264）
- 分辨率：1280×720 或 1920×1080
- 时长：10–30 秒可循环
- 单文件建议 < 10MB

### 音频

- 格式：`.mp3`
- 章节 BGM 与角色语音分文件存放
- 公开部署须注意版权；自用本地预览风险较低

### 图片

- 格式：`.jpg` / `.png`
- 角色卡：≥ 400×400
- 个人照：竖图更佳（About 拍立得样式）

### 路径

- 一律使用以 `/` 开头的 public 路径，如 `/videos/ch01.mp4`
- 文件名用小写 + 数字编号，避免中文文件名

## 4. 代码规范

- **优先改 `config.ts`**，不扩 scope 改无关组件
- 硬编码标题改动限于 `Companions.tsx`、`Seasons.tsx`、`TimeSlider.tsx`，保持最小 diff
- 新增章节只改 `config.ts` 的 `chapters` 数组，不手动改布局逻辑
- 遵循项目既有风格：TypeScript、Tailwind、`@/` 路径别名
- 不提交 `.env`、密钥、个人隐私原始素材到公开仓库（若后续开源）

## 5. Git 规范

- 仅在用户明确要求时 commit / push
- 提交前确认未包含敏感个人信息（若用户有顾虑）
- 提交信息聚焦「为什么」，如：`docs: add life story project documentation`

## 7. 命名与主题色建议

可按人生阶段分配色系，填入 `color` / `glassBg`：

| 阶段 | color 参考 | 气质 |
|------|------------|------|
| 小学 | `#FAD0C4` | 暖黄、童真 |
| 初中 | `#CE93D8` | 紫粉、好奇 |
| 高中 | `#64B5F6` | 蓝、紧张与希望 |
| 大学 | `#81C784` | 绿、青春 |
| 研究生 | `#FFB74D` | 橙、奔波与坚持 |
| 职场 | `#90A4AE` | 灰蓝、沉稳 |
| 特殊经历 | `#B39DDB` | 紫、内省与力量 |

`glassBg` 使用对应 `color` 的 rgba，透明度约 0.10–0.15。
